LimeTorrent REST API Documentation

Base URL: http://[host]:[port] (default: http://127.0.0.1:5000)
Authentication: Pass Lime-API-Key: <key> header or use session cookie (WebUI login)
Content-Type: application/json for JSON bodies; multipart/form-data for file uploads

⚡ Quick Start

Get up and running in under a minute. Replace YOUR_API_KEY with the key shown on startup.

# 1. Health check
curl http://127.0.0.1:5000/health

# 2. Add a magnet link
curl -s -X POST http://127.0.0.1:5000/add/magnet \
  -H "Content-Type: application/json" \
  -H "Lime-API-Key: YOUR_API_KEY" \
  -d '{"magnet":"magnet:?xt=urn:btih:INFOHASH"}'

# 3. List all torrents
curl -H "Lime-API-Key: YOUR_API_KEY" http://127.0.0.1:5000/list

🔐 Authentication

LimeTorrent supports two authentication methods:

  • API Key — Pass the Lime-API-Key header. Ideal for scripts and automation. API keys have a level: admin (full access) or user (respects account permissions).
  • Session Cookie — Log in via /webui/login. Only one active session per account is permitted at a time.
Admin credentials are RAM-only. The admin password and API key are never written to any file — they are reconstructed from CLI args / environment variables on every startup. To change the admin password, restart with --webui-pass NEW_PASSWORD or set WEBUI_PASS. If you lose your credentials, visit GET /info (no auth required).
POST /webui/login
Authenticate and receive a session cookie. Only one active session per account is allowed; a second login from a different client is rejected with 409 until the first session is logged out.

Request Body (JSON)

FieldTypeRequiredDescription
usernamestring✓Account username
passwordstring✓Account password (admin password shown on startup)

Response (200)

{"ok": true, "role": "admin", "permissions": ["add_torrent", ...]}

Response (401 — wrong credentials)

{"error": "Invalid credentials"}

Response (409 — already logged in elsewhere)

{"error": "This account is already logged in from another session. Please logout first."}

Response (429 — too many attempts)

{"error": "Too many attempts. Try again in 300s"}
POST /webui/logout
Logout and release the active session slot, allowing a new login from any client.
{"ok": true}
⚠ Login is rate-limited: after 3 failed attempts from the same IP, the account is locked for 5 minutes. Only one active session per account is allowed at a time. Use POST /users/<username>/kick (admin) to force-clear a stuck session.

⚠️ Error Handling

All error responses use standard HTTP status codes with a JSON body:

{"error": "Human-readable description"}
CodeMeaning
200Success — body contains {"ok": true, ...}
400Bad request — missing or invalid parameters
401Unauthorized — missing or invalid API key / session
403Forbidden — permission denied or HTTP download disabled
404Torrent not found
409Conflict — torrent already exists
429Rate limited (login lockout)
500Internal server error

🎮 Torrent Control

All control endpoints require authentication. Replace {hash} with the torrent's info hash.

POST/pause/{hash}
Pause a torrent. The torrent remains managed (can auto-resume).
curl -s -X POST http://127.0.0.1:5000/pause/INFOHASH \
  -H "Lime-API-Key: YOUR_API_KEY"
curl -s -X POST http://127.0.0.1:5000/pause/INFOHASH ^
  -H "Lime-API-Key: YOUR_API_KEY"
import requests
r = requests.post("http://127.0.0.1:5000/pause/INFOHASH",
                  headers={"Lime-API-Key": "YOUR_API_KEY"})
print(r.json())
POST/resume/{hash}
Resume a paused or stopped torrent.
POST/stop/{hash}
Stop a torrent explicitly (auto_managed=off). State becomes completed if progress=100%, else stopped.
POST/stop/file
Stop a torrent identified by uploading its .torrent file (multipart, field: torrent).
DELETE/remove/{hash}
Remove a torrent. Pass ?delete_files=1 to also delete the downloaded data from disk.
# Remove torrent only
curl -s -X DELETE http://127.0.0.1:5000/remove/INFOHASH \
  -H "Lime-API-Key: YOUR_API_KEY"

# Remove torrent AND delete files on disk
curl -s -X DELETE "http://127.0.0.1:5000/remove/INFOHASH?delete_files=1" \
  -H "Lime-API-Key: YOUR_API_KEY"
curl -s -X DELETE "http://127.0.0.1:5000/remove/INFOHASH?delete_files=1" ^
  -H "Lime-API-Key: YOUR_API_KEY"
import requests
# delete_files=True removes data from disk
r = requests.delete(
    "http://127.0.0.1:5000/remove/INFOHASH",
    headers={"Lime-API-Key": "YOUR_API_KEY"},
    params={"delete_files": 1}
)
print(r.json())
DELETE/remove/file
Remove a torrent identified by its .torrent file (multipart, field: torrent).
POST/recheck/{hash}
Force a piece-by-piece recheck of the downloaded data.
POST/announce/{hash}
Force an immediate re-announce to all trackers.

⬇️ Download Completed Files

HTTP file download must be enabled server-wide via --http-download CLI flag or HTTP_DOWNLOAD=1 env var. Additionally, the user/API key must have the download_file permission. The server admin can toggle this feature at runtime via POST /settings/http-download.
GET /download/file/{api_key}/{hash}/{path/to/file}
Download a completed file from a torrent directly from the server. The API key is embedded in the URL path, not as a query parameter.

Requirements:
  • HTTP download must be enabled server-wide (--http-download)
  • The API key must be valid and have download_file permission
  • The target file must be 100% downloaded (progress ≥ 100%)
  • Torrent must be in completed or seeding state

URL Parameters

ParameterDescription
api_keyYour API key (shown on server startup or in WebUI Settings)
hashInfo hash of the torrent
path/to/fileFull path of the file inside the torrent (URL-encoded)

Example

# Download a file from a torrent (file must be 100% complete)
curl -O "http://127.0.0.1:5000/download/file/YOUR_API_KEY/INFOHASH/MyMovie.mkv"
import requests

url = "http://127.0.0.1:5000/download/file/YOUR_API_KEY/INFOHASH/MyMovie.mkv"
resp = requests.get(url, stream=True)
with open("MyMovie.mkv", "wb") as f:
    for chunk in resp.iter_content(chunk_size=8192):
        f.write(chunk)

🌱 Create & Seed

POST/create
Create a .torrent file from local data on the server. Returns the .torrent file as a binary download.

Request Body (JSON)

FieldTypeRequiredDescription
pathstring✓Absolute path to file or directory on server
trackersarray—List of tracker announce URLs — supports flat list, per-tier objects, or nested tier arrays (see below)
trackerstring—Single tracker URL (legacy / backward-compat — prefer trackers)
piece_sizeinteger—Piece size in bytes (default: auto-selected by libtorrent)
commentstring—Optional comment embedded in the torrent metadata
privateboolean—Set private flag (disables DHT/PEX — required by most private trackers)

Tracker Format Options

The trackers field accepts three input formats (can be mixed): 

// Flat list of URL strings — all placed in tier 0 (announce order is random within tier)
{
  "path": "/data/MyContent",
  "trackers": [
    "udp://tracker.opentrackr.org:1337/announce",
    "udp://open.stealth.si:80/announce",
    "https://tracker.tamersunion.org:443/announce"
  ]
}
// Per-tier objects — {"url": "...", "tier": N}
// Lower tier number = higher priority (per BitTorrent spec)
{
  "path": "/data/MyContent",
  "trackers": [
    {"url": "udp://primary-tracker.example:1337/announce", "tier": 0},
    {"url": "udp://backup-tracker.example:1337/announce",  "tier": 1},
    {"url": "https://fallback.example:443/announce",       "tier": 2}
  ]
}
// Nested list — each sub-array is one tier; array index = tier number
{
  "path": "/data/MyContent",
  "trackers": [
    ["udp://primary1.example:1337/announce", "udp://primary2.example:1337/announce"],
    ["udp://backup.example:1337/announce"],
    ["https://fallback.example:443/announce"]
  ]
}

Toggle super-seeding mode for a specific torrent. Super-seeding sends each piece only once, maximizing swarm spread.

Response

{"ok": true, "super_seeding": true}

done

"""LimeTorrent Python client example using `requests`."""
import time
import requests

BASE    = "http://127.0.0.1:5000"
API_KEY = "YOUR_API_KEY"
HEADERS = {"Lime-API-Key": API_KEY}


def check_health():
    r = requests.get(f"{BASE}/health")
    print("Health:", r.json())


def add_magnet(magnet: str, save_path: str = None):# -> str | None:
    payload = {"magnet": magnet}
    if save_path:
        payload["save_path"] = save_path
    r = requests.post(f"{BASE}/add/magnet", headers=HEADERS, json=payload)
    data = r.json()
    if r.ok and data.get("ok"):
        print(f"Added torrent: {data['hash']}")
        return data["hash"]
    print("Error:", data.get("error"))
    return None


def wait_for_done(info_hash: str, poll_sec: int = 5):
    """Poll until the torrent is seeding or stopped."""
    while True:
        r = requests.get(f"{BASE}/status/{info_hash}", headers=HEADERS)
        t = r.json()
        progress = t.get("progress", 0)
        state    = t.get("state", "?")
        print(f"  {state:12} {progress:5.1f}%  ↓{t.get('download_speed',0)//1024} KB/s")
        if state in ("seeding", "completed", "stopped"):
            break
        time.sleep(poll_sec)


def set_speed_limits(upload_bps: int = 0, download_bps: int = 0):
    r = requests.post(f"{BASE}/limit/global", headers=HEADERS,
                      json={"upload_limit": upload_bps, "download_limit": download_bps})
    print("Limits set:", r.json())


def download_file(info_hash: str, file_path: str, output_name: str = None):
    """Download a completed file from a torrent."""
    url = f"{BASE}/download/file/{API_KEY}/{info_hash}/{file_path}"
    resp = requests.get(url, stream=True)
    if resp.status_code == 200:
        out_name = output_name or file_path.split("/")[-1]
        with open(out_name, "wb") as f:
            for chunk in resp.iter_content(chunk_size=8192):
                f.write(chunk)
        print(f"Downloaded: {out_name}")
    else:
        print(f"Error: {resp.status_code} - {resp.text}")


def remove_torrent(info_hash: str, delete_files: bool = False):
    params = {"delete_files": 1} if delete_files else {}
    r = requests.delete(f"{BASE}/remove/{info_hash}", headers=HEADERS, params=params)
    print("Removed:", r.json())


if __name__ == "__main__":
    check_health()
    h = add_magnet("magnet:?xt=urn:btih:INFOHASH&dn=TestFile")
    if h:
        set_speed_limits(upload_bps=1_048_576)   # 1 MB/s upload cap
        wait_for_done(h)
        # Download a file from the torrent (if complete)
        download_file(h, "MyMovie.mkv")
        remove_torrent(h)

import java.net.URI;
import java.net.http.*;
import java.nio.file.*;
import java.time.Duration;

public class LimeTorrentClient {

    private static final String BASE    = "http://127.0.0.1:5000";
    private static final String API_KEY = "YOUR_API_KEY";

    private final HttpClient client = HttpClient.newBuilder()
        .connectTimeout(Duration.ofSeconds(10))
        .build();

    private HttpResponse<String> get(String path) throws Exception {
        HttpRequest req = HttpRequest.newBuilder()
            .uri(URI.create(BASE + path))
            .header("Lime-API-Key", API_KEY)
            .GET().build();
        return client.send(req, HttpResponse.BodyHandlers.ofString());
    }

    private HttpResponse<String> post(String path, String json) throws Exception {
        HttpRequest req = HttpRequest.newBuilder()
            .uri(URI.create(BASE + path))
            .header("Content-Type", "application/json")
            .header("Lime-API-Key", API_KEY)
            .POST(HttpRequest.BodyPublishers.ofString(json))
            .build();
        return client.send(req, HttpResponse.BodyHandlers.ofString());
    }

    public void downloadFile(String infoHash, String filePath, String outputPath) throws Exception {
        String url = String.format("%s/download/file/%s/%s/%s", BASE, API_KEY, infoHash, filePath);
        HttpRequest req = HttpRequest.newBuilder()
            .uri(URI.create(url))
            .GET().build();
        HttpResponse<Path> res = client.send(req, HttpResponse.BodyHandlers.ofFile(Path.of(outputPath)));
        System.out.println("Downloaded: " + res.body());
    }

    public static void main(String[] args) throws Exception {
        LimeTorrentClient lt = new LimeTorrentClient();

        // Health
        System.out.println("Health: " + lt.get("/health").body());

        // Add magnet
        String magnetJson = "{\"magnet\":\"magnet:?xt=urn:btih:INFOHASH\"}";
        System.out.println("Add: " + lt.post("/add/magnet", magnetJson).body());

        // List
        System.out.println("List: " + lt.get("/list").body());

        // Set speed limits (1 MB/s up, 5 MB/s down)
        String limitsJson = "{\"upload_limit\":1048576,\"download_limit\":5242880}";
        System.out.println("Limits: " + lt.post("/limit/global", limitsJson).body());

        // Download a file (if complete)
        // lt.downloadFile("INFOHASH", "MyMovie.mkv", "downloaded_movie.mkv");
    }
}

func doGet(path string) ([]byte, error) {
    req, _ := http.NewRequest("GET", base+path, nil)
    req.Header.Set("Lime-API-Key", apiKey)
    res, err := http.DefaultClient.Do(req)
    if err != nil { return nil, err }
    defer res.Body.Close()
    return io.ReadAll(res.Body)
}

func doPost(path string, payload any) ([]byte, error) {
    body, _ := json.Marshal(payload)
    req, _ := http.NewRequest("POST", base+path, bytes.NewBuffer(body))
    req.Header.Set("Content-Type", "application/json")
    req.Header.Set("Lime-API-Key", apiKey)
    res, err := http.DefaultClient.Do(req)
    if err != nil { return nil, err }
    defer res.Body.Close()

}

    if err != nil { return err }
